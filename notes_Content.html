  • Updating Existing Content
    • Combining the JavaScript and the DOM
      • Will need strong understanding of the following things from the previous lesson
        • DOM Selection Model
        • .getElementByID()
        • .getElementsByClassName()
        • .querySelector()
        • Interfaces
        • Node
        • Element
    • .innerHTML
      • Every Element inherits properties and methods from the Element Interface
      • Therefore, every Element has an .innerHTML property
      • Represents the markup of the element's content
      • Can be used to:
        • get an element's (and all its descendants'!) HTML content
        • set an element's HTML content
      • Returns - a string, a DOMString
      • Sets or Returns the HTML content inside the selected element (between the tags)
      • If you want to update an element, including its HTML, use the .innerHTML property
    • .outerHTML
      • represents the HTML element itself, as well as its children
    • .textContent
      • If we just want the text content of an element, use .textContent
      • Can be used to:
        • set the text content of an element and all its descendants
        • return the text content of an element and all its descendants
      • https://developer.mozilla.org/en-US/docs/Web/API/Node/textContent
      • ONLY displays the actual text shown on the site for this element
      • Passing new text through .textContentwill not display the content as HTML, instead, all displayed as plain text
    • .innerText
      • Similar to .textContent, .innerText property can be used to get/set an element's text content, but with diffs
      • .innerText is a little bit trickier
      • returns all visible text of the element, e.g. only what will be shown on the actual page
      • Adheres to hide/show, capitalization, and other items of CSS
    • For setting HTML content for an element:
      • .innerHTML
  • Adding New Content
    • .createElement()
      • A method on the document object
      • Ex. - document.createElement('p') creates new p element
      • Creates the element, but doesn't add it to the DOM
    • .createTextNode()
      • Just like .createElement(), .createTextNode() allows you to create new text nodes
    • .appendChild()
      • Can be used to add an element to the page (including it in the DOM)
      • To do so, it needs to be called on another element, not the document object!
      • Ex. // create a brand new element
        const newSpan = document.createElement('span');
        // select the first (main) heading of the page
        const mainHeading = document.querySelector('h1');
        // add the the element as the last child element of the main heading
        mainHeading.appendChild(newSpan);
      • Needs an Element!!!
      • Can't be called on a document object - will error out
      • Warning - will move an element from its current position to the new (newly designated) position
    • .insertAdjacentHTML()
      • Allows us to insert a new element at a location other at the very end of the parent element
      • Has two arguments:
        • the location of the HTML
          • beforebegin - as a previous sibling
          • afterbegin - as the first child
          • beforeend - as the last child
          • afterend - as a following sibling
          • Clarification:
            !-- beforebegin -->
            p>
            !-- afterbegin -->
            Existing text/HTML content !-- beforeend -->
            p>
            !-- afterend -->
          • the HTML text that is going to be inserted
        • Needs to be called on a parent element
    • Removing Page Content
      • .removeChild()
        • Used to remove a child element
        • Essentially the opposite of .appendChild()
        • Requires:
          • a parent element
          • the child element to be removed
          • ex. parent-element.removeChild(Child-to- remove)
        • Drawback - requires access to the parent element, but can use the .parentElement to work around this!
        • Ex.:
          const mainHeading = document.querySelector('h1');
          mainHeading.parentElement.removeChild(mainHeading);
      • .remove()
        • directly removes what element is in question
      • .firstChild
        • Might return whitespace (if there is any) to preserve the formatting of the underlying code
      • .firstElementChild
        • Returns the first element of the parent element with content
      • .parentElement
        • Returns the parent element of the element in question
    • Styling Page Content
      • CSS Specificity
        • According to MDN, specificity is: the means by which browsers decide which CSS property values are the most relevant to an element and, therefore, will be applied
      • .style.property
        • Able to access an elements style property directly
        • Ex. - const mainHeading = document.querySelector('h1');
          mainHeading.style.color = 'red';
        • Can only modify one CSS style at a time
        • variable.style will list all available CSS styles, whether empty or not, for that element
      • .style.cssText()
        • Able to modify multiple styles at once
        • style.property can access one, CSSText can access many
        • Ex. - const mainHeading = document.querySelector('h1');
          mainHeading.style.cssText = 'color: red; background-color: orange; font-size: 3.5em;')
      • .setAttribute()
        • Able to set specific attributes
        • Ex. - const mainHeading = document.querySelector('h1');
          mainHeading.setAttribute('style', 'color: red; background-color: orange; font-size: 3.5em;')
        • Not just for styling, can also be used to set things like IDs
      • .className
        • Returns a space-separated string of classes
        • Can then be converted to an array with the .split() method
        • Setting a new property with this method erases the previous properties
      • .classList
        • MOST HELPFUL OF THE BUNCH
        • Returns a list of all classes specific to that element
        • Has a number of properties of its own:
          • .add() - add a class to the list
          • .remove() - remove a class from the list
          • .toggle() - add the class if it doesn't exist, or remove it if it does exist
          • .contains() - returns a boolean based on if the class exists in the list or not
